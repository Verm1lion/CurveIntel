# Vendor-agnostic UTM CSV parsing: a field guide for seven tensile-test instruments

**The core finding is that all seven tensile-software exports are customizable by design, not rigid schemas — which means a robust Python parser cannot rely on fixed row offsets or column orders and must use header-sniffing plus multilingual column-name aliasing.** Only Shimadzu Trapezium and Instron Bluehill have well-documented structures and actively maintained open-source parsers; ZwickRoell's native binary is reverse-engineered via `zs2decode`, and the remaining four vendors (Tinius Olsen, MTS, DEVOTRANS, Hegewald & Peschke) publish export capability without a concrete schema. Below, every fact is labeled **[DOC]** (documented in vendor literature or a working parser) vs **[GUESS]** (inferred from industry convention) so you can calibrate production confidence.

The report is organized by vendor. Each primary vendor covers all eight categories (A-H) with Python `column_map` and fingerprint regex. Bonus vendors get compact summaries. A final synthesis section recommends a detection/parsing architecture.

## ZwickRoell testXpert III (and II)

ZwickRoell produces the most complex file landscape: a proprietary binary plus three ASCII variants. **The native `.zs2`/`.zp2` format is binary and undocumented by Zwick, but has been reverse-engineered.** The `.TRA` extension is a raw ASCII curve export, and testXpert III's "Export Editor" additionally writes Excel-compatible CSV/XML. Any parser must therefore first identify which of these three families a file belongs to.

**A. Header structure [partial DOC]:** The Export Editor in testXpert III produces multi-section output: a parameter block, a results/statistics block, and an X-Y curve block with a user-configurable number of samples. Delimiter is typically **tab (`\t`)** for `.TRA` and **comma or semicolon** for CSV depending on locale. A German-locale install commonly writes semicolon-separated files with comma decimals. The exact metadata row count is variable; parsers must scan until the numeric table start.

**B. Default column names (ISO 6892-1 tensile template):** Localized per UI language. testXpert supports 11+ UI languages and export language can differ from UI language, so both German and English appear frequently in the wild.

| German | English | Physical quantity |
|---|---|---|
| Standardkraft | Standard force | Load (N or kN) |
| Standardweg | Standard travel | Crosshead displacement (mm) |
| Traverse / Traversenweg | Crosshead / Extension | Machine displacement (mm) |
| Dehnung | Tensile strain | Strain (%) |
| Spannung / Zugspannung | Stress / Tensile stress | Engineering stress (MPa) |
| Zeit / Prüfzeit | Time / Test time | Elapsed time (s) |
| Extensometer-Dehnung | Extensometer strain | Clip-on extensometer (%) |
| Probenbreite | Specimen width | mm |
| Probendicke | Specimen thickness | mm |
| Anfangsmesslänge (L₀) | Initial gauge length | mm |

**C. Metadata content [DOC]:** Typical key-value pairs in the header block: *Prüfmaschine/Machine*, *Prüfnorm* (test standard, e.g. ISO 6892-1), *Prüfdatum/Test date*, *Prüfer/Operator*, *Probenbezeichnung/Specimen ID*, *Prüfgeschwindigkeit/Test speed (mm/min)*, *L₀*, *Querschnittsfläche/Cross-section*. Delivered as one key per line, separated by the file's field delimiter.

**D. Default units [DOC]:** Force in **N** (auto-switched to **kN** for large load cells), displacement in **mm**, stress in **MPa**, strain in **%**, time in **s**.

**E. Decimal separator [DOC — locale dependent]:** German/European installs use **comma (`,`)**; English installs use **dot (`.`)**. This is directly coupled to the delimiter choice — German exports almost always use `;` + `,` to avoid ambiguity.

**F. Encoding [GUESS, strong industry convention]:** **Windows-1252** (CP1252) is the overwhelming default on German/European testXpert installations because of characters like `ä`, `ö`, `ü`, `ß` and the `²` in `N/mm²`. UTF-8 is offered in newer testXpert III builds but not default. No BOM.

**G. testXpert II vs III differences [DOC]:** testXpert II introduced direct-to-Excel export in v3.1; testXpert III replaced it with the more flexible "Export Editor" (user-designed templates including CSV, Excel, XML, PDF). The `.zs2` internal format is shared between II and III but III adds additional chunks. `zs2decode` was reverse-engineered against testXpert II v3.1 and is known to work but may miss newer III-only fields — flag for production use.

**H. Fingerprints for auto-detection [DOC]:** Strong signals are the literal strings `testXpert`, `ZwickRoell`, `Zwick Roell`, `Zwick/Roell`, the German header `Standardkraft` or `Standardweg`, and the NI DataPlugin identifier `TXpert`. The `.TRA` extension is itself a strong hint (shared historically with Instron but overwhelmingly Zwick today).

## Instron Bluehill Universal (and Bluehill 3)

**Bluehill is the most configurable exporter of the three primaries — virtually every aspect of the CSV is user-controlled, including delimiter, decimal mark, encoding, and column selection.** This means auto-detection must rely on filename conventions and a few structural invariants rather than fixed headers.

**A. Header structure [DOC]:** Raw exports live in a sibling folder to the native sample: `<sample>.is_tens_RawData/Specimen_RawData_<n>.csv`, with an accompanying `<sample>.is_tens_Results.csv`. Inside each raw CSV: optional multi-line metadata block, then **one row of column names**, then **one row of units in parentheses**, then numeric data. Users can individually disable the metadata block, the column-names row, or the units row in the export template.

**B. Default column names (English tensile template) [DOC]:** Five canonical channels appear in nearly every real-world Bluehill file: `Time (s)`, `Extension (mm)`, `Load (N)` or `Load (kN)`, `Tensile strain (Extension) (mm/mm)` or `(%)`, `Tensile stress (MPa)`. Extensometer installations add `Tensile strain (Strain 1)`. **Raw Data export** gives the time series; **Results export** instead gives one row per specimen with computed quantities like `Modulus (E-modulus)`, `Tensile strength`, `Yield strength (0.2%)`, `Strain at break`.

**C. Metadata content [DOC]:** Specimen label, Sample name, Test date, Method name, Operator, Specimen dimensions (width, thickness, gauge length), Test speed, Machine model, Serial number. User-selectable inputs are also appendable.

**D. Units in parentheses [DOC]:** Units are written on the line below the channel names, wrapped in parentheses: `(s)`, `(mm)`, `(N)`, `(kN)`, `(MPa)`, `(%)`, `(mm/mm)`.

**E. Decimal separator [DOC — fully locale-driven]:** The export dialog exposes "Decimal symbol: system default / period / comma". A German install defaults to comma, with column separator typically flipped to semicolon or tab to avoid ambiguity.

**F. Encoding [DOC]:** User-selectable: Default (system), ASCII, **UTF-16**, UTF-8. UTF-16 with BOM is surprisingly common from Bluehill Universal, which breaks naïve `pd.read_csv(encoding="utf-8")` calls — always sniff BOMs first.

**G. Bluehill 3 vs Universal [DOC]:** File/folder conventions (`.is_tens`, `.im_tens`, `.id_tens`, `Specimen_RawData_n.csv`, `_Results.csv`) are shared across Bluehill 2, 3, and Universal. Universal's Export Editor is more flexible (additional separator choices, output layout horizontal/vertical, value-quoting character customization).

**H. Fingerprints [DOC]:** Filename pattern `Specimen_RawData_\d+\.csv` alongside `_Results.csv` is the strongest single indicator. Metadata keys like `Sample name`, `Specimen label`, `Method`, `Bluehill`, or the `.is_tens/.im_tens/.id_tens` neighbors in the folder tree are also reliable. The literal string `Instron` appears in some metadata headers.

## Shimadzu Trapezium X / TrapeziumLite X

**The `snpl` Python package is the single most authoritative open-source reference for Trapezium CSVs**, confirming Shift-JIS encoding, comma delimiter default, and a multi-section layout with Japanese key-value metadata followed by column-name and unit rows.

**A. Header structure [DOC]:** Comma-delimited by default (`dialect='excel'`), with tab-delimited TSV as a user-selectable alternative. Multi-section layout: a variable-length metadata block (typically 10-40 rows), one batch/specimen header per test piece introduced by `名前` (Name), then column-name row, then unit row, then numeric data. For Cycle programs each cycle carries a `Cycle ID` marker.

**B. Default column names [DOC for Japanese, DOC for English product strings]:** Japanese is the original set; English headers appear under an English UI.

| Japanese | English | Quantity |
|---|---|---|
| 試験力 | Test Force / Force | Load (N, kN) |
| ストローク | Stroke / Crosshead | Displacement (mm) |
| つかみ具間変位 | Grip Separation | Grip-to-grip (mm) |
| 伸び / 伸び計変位 | Elongation / Extensometer | Extension (mm) |
| 応力 / 真応力 | Stress / True Stress | MPa |
| ひずみ / 真ひずみ | Strain / True Strain | % |
| 時間 | Time | s |
| 試験片 | Specimen | label |
| 厚さ / 幅 / 標点間距離 | Thickness / Width / Gauge Length | mm |

**C. Metadata [DOC]:** Key-value pairs `試験条件ファイル名/Method Name`, `試験者/Operator`, `試験日時/Test Date` (`YYYY/MM/DD HH:MM:SS`), `試験機/Machine` (e.g. `AGS-X 10kN`), `ロードセル/Load Cell`, `試験速度/Test Speed`, and per-batch specimen dimensions.

**D. Default units [DOC]:** N/kN, mm, MPa (= N/mm²), %, s. Units on the row immediately beneath the column names.

**E. Decimal separator [DOC]:** **Dot (`.`) even in Japanese locale** — confirmed by the `snpl` parser using plain float parsing. Comma decimals in European installs are theoretically possible but unconfirmed.

**F. Encoding [DOC]:** **Shift-JIS (CP932) with no BOM** on Japanese installs, hard-coded as the `snpl` default. Western installs likely write ASCII/Windows-1252. CRLF line endings.

**G. Variants [DOC]:** Trapezium X supports four test programs (Single, Cycle, Control, Texture), each with a slightly different layout; TrapeziumLite X is a subset missing Cycle/Control/Texture. Machine models AGS-X, AG-IS, AGX-V, AGX-V2, AUTOGRAPH all feed the same Trapezium X / X-V software — the only differentiator in the CSV is the `Machine` metadata string. Trapezium 2 (older, EZ-L/EZ-S series) shares the same basic layout.

**H. Fingerprints [DOC]:** Literal strings `TRAPEZIUM`, `Trapezium X`, `Shimadzu`, `Autograph`; Japanese keys `試験力`, `ストローク`, `名前`, `試験条件`, `試験機`; Cycle export's `Cycle ID` column; machine strings beginning `AGS-`, `AG-`, `AGX-`, `EZ-`, `AUTOGRAPH`.

## Recommended architecture and key takeaways

The central design implication is that **hard-coded `skiprows=N` is wrong for every vendor here.** Every major exporter allows the user to toggle the metadata block, column-name row, and unit row independently. Build auto-detection as a two-stage pipeline: first a **vendor fingerprint sniff** over the first ~40 decoded lines using the regex set above, then a **header-row discovery** pass that scans line by line for the first row whose tokenized cells match ≥2 entries in the vendor's column map, treating the next line as the units row if parenthesized or bracketed unit strings are present.

For **encoding detection**, try the chain `utf-8-sig → utf-16 → cp932 → cp1252 → cp1254 → latin-1` — Bluehill's UTF-16 BOM output, Shimadzu's Shift-JIS, Zwick/Hegewald's Windows-1252, and DEVOTRANS's potential Windows-1254 each break naïve UTF-8 assumptions. For **delimiter and decimal**, use `csv.Sniffer` on the first data row, but override the sniffer if the column-name row contains more field separators than the data row (a classic Bluehill-German-locale failure mode). Be prepared for `;` + `,`-decimal (German locale), `\t` + `.`-decimal (Shimadzu JP and Zwick TRA), and `,` + `.`-decimal (US defaults).

For **column mapping**, flatten all seven vendor dicts into a single master lookup keyed case-insensitively, with parenthesized unit suffixes stripped before matching (`Load (kN)` → `load`). Preserve the original unit string separately so a downstream converter can rescale `kN → N`, `mm/mm → %`, and `N/mm² → MPa`.

**The largest remaining risk is the four bonus vendors plus Zwick's exact `.TRA` header layout — none of these have publicly posted example files that I could verify. Production use should start with a small corpus of real exports from each target lab before locking any schema.** The three primary vendors can be shipped with high confidence using the fingerprints and column maps above, calibrated against `snpl` (Shimadzu), `yadbor/bluer` (Instron), and `zs2decode` + `extrudion` (Zwick) as ground-truth references.
