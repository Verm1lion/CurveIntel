ZwickRoell, Instron, Shimadzu, MTS ve Tinius Olsen gibi küresel pazarın dominant cihaz üreticileri (vendor), kendi tescilli kontrol ve analiz yazılımları üzerinden verileri dışa aktarırken birbirinden tamamen farklı CSV (Comma-Separated Values) veya ASCII formatları, yapısal hiyerarşiler, karakter kodlamaları (encoding) ve yerelleştirme (localization) kuralları kullanmaktadır. Bu yapısal çeşitlilik, veritabanlarına veya veri göllerine (data lakes) otomatik veri akışı sağlamayı hedefleyen sistemler için aşılması gereken çok katmanlı bir mühendislik problemi yaratmaktadır.Makine öğrenmesi modelleri, yapay zeka tabanlı malzeme keşfi algoritmaları, istatistiksel süreç kontrolü (SPC) sistemleri ve büyük veri platformları, doğaları gereği doğrudan yapılandırılmış, temizlenmiş ve homojen veri setlerine ihtiyaç duyarlar. Ancak, bir UTM'den alınan ham CSV dosyaları; test cihazının anlık kalibrasyon durumu, sensör konfigürasyonları, ekstensometre (extensometer) çıkarılma anı, ön yükleme (preload) periyotları ve testin gerçekleştirildiği coğrafi bölgenin işletim sistemi ayarlarına (ondalık ve binlik ayraçlar) bağlı olarak son derece dinamik ve kuralsız bir yapı sergilemektedir. Üretici-bağımsız (vendor-agnostic) bir Python ayrıştırıcısı (parser) geliştirmek, her bir dışa aktarım formatının yapısal anatomisini derinlemesine ve analitik bir bakış açısıyla anlamayı gerektirir. Gerçekleştirilen bu mimari ve yapısal analiz; meta veri (metadata) bloklarının algılanması, kolon isimlerinin çoklu dil desteğiyle sözlükler halinde eşleştirilmesi, ondalık/binlik ayraçların lokalizasyon bağımsız şekilde dönüştürülmesi ve sensör limitlerine ulaşılması gibi fiziksel test anomalilerinin dijital formattaki izlerinin tespiti aşamalarını kapsamaktadır.Aşağıdaki kapsamlı analiz, endüstrinin en yaygın kullanılan cihazları için detaylı bir veri yapısı topolojisi sunmakta ve Python tabanlı otomatik algılama (fingerprinting) ve ayrıştırma mekanizmaları için gerekli olan kurallar bütününü inşa etmektedir. import pandas as pd
from abc import ABC, abstractmethod

class BaseUTMParser(ABC):
    """
    Tüm üretici-özel ayrıştırıcıların (parser) türetileceği Temel Sınıf.
    Her alt sınıf kendi identify (kimlik doğrulama) ve parse_data mantığını uygular.
    """
    @abstractmethod
    def identify(self, header_text: str) -> bool:
        """Dosyanın ilk 2000 karakterinden üreticiyi tespit eder."""
        pass
        
    @abstractmethod
    def parse_data(self, file_path: str) -> pd.DataFrame:
        """Ham dosyayı standartlaştırılmış bir Pandas DataFrame'ine çevirir."""
        pass

class ZwickRoellParser(BaseUTMParser):
    def identify(self, header_text: str) -> bool:
        pattern = re.compile(r'(Machine name|Maschinenname|My Zwicki|Standardkraft|Standard force)', re.IGNORECASE)
        return bool(pattern.search(header_text))
        
    def parse_data(self, file_path: str) -> pd.DataFrame:
        # DACH lokalizasyonu: Liste ayırıcı noktalı virgül, ondalık virgül.
        # cp1252 (Windows-1252) encoding German locale için güvenli limandır.
        # Meta veriyi atlamak için skiprows dinamik hesaplanmalıdır, örnekte statik verilmiştir.
        return pd.read_csv(file_path, sep=';', decimal=',', encoding='cp1252', skiprows=20)

class InstronBluehillParser(BaseUTMParser):
    def identify(self, header_text: str) -> bool:
        pattern = re.compile(r'("Table:"\s*,\s*\d+|"Test Type"\s*,\s*".+")', re.IGNORECASE)
        return bool(pattern.search(header_text))
        
    def parse_data(self, file_path: str) -> pd.DataFrame:
        # Instron'da "Table:,1" satırına kadar olan metadata Python üzerinden atlanabilir.
        # UTF-8 kullanmak Instron için optimaldir.
        return pd.read_csv(file_path, sep=',', encoding='utf-8')

class ShimadzuTrapeziumParser(BaseUTMParser):
    def identify(self, header_text: str) -> bool:
        pattern = re.compile(r'(TRAPEZIUM|Test Force|試験力|Stroke|ストローク)', re.IGNORECASE)
        return bool(pattern.search(header_text))
        
    def parse_data(self, file_path: str) -> pd.DataFrame:
        # Shimadzu'nun Japonca karakter kodlama krizi için try-except bloğu hayat kurtarır.
        try:
            df = pd.read_csv(file_path, sep=',', encoding='shift_jis')
        except UnicodeDecodeError:
            df = pd.read_csv(file_path, sep=',', encoding='utf-8')
        return df

class TiniusOlsenParser(BaseUTMParser):
    def identify(self, header_text: str) -> bool:
        # '#' karakteri ile başlayan metadata satırları Tinius Olsen karakteristiğidir.
        return header_text.startswith('#')
        
    def parse_data(self, file_path: str) -> pd.DataFrame:
        # comment='#' parametresi sayesinde tüm metadata otomatik bypass edilir.
        return pd.read_csv(file_path, sep=',', comment='#', encoding='utf-8')

Veri Normalizasyonu ve SonuçGeliştirilen bu mimari, yalnızca ayrıştırma (parsing) işlemi yapmakla kalmaz; aynı zamanda makine öğrenmesi mühendislerinin veriyi normalleştirmesi (normalization) için güvenilir bir yapı bloğu oluşturur. Mekanik test verilerindeki en kritik analitik sorunlardan biri, testin ilerleyen aşamalarında donanım tabanlı değişikliklerin yazılıma "zıplama" (jump) veya "ölü zaman" (dead time) olarak yansımasıdır.Örneğin, Instron ve Zwick sistemleri yüksek maliyetli ve hassas optik/mekanik ekstensometreleri, numunenin kopma anında (break point) hasar görmemesi için "Extensometer Removal Criteria" (Ekstensometre Çıkarma Kriteri) ile devre dışı bırakır. Bu an, CSV dosyası içerisinde birim uzama (strain) kolonunda bir algoritmik kırılma yaratır; çünkü sensör okumasından çıkılıp motordan (crosshead) gelen enkoder verisine geçilmiştir. Veri madenciliği algoritmaları için bu geçiş, CSV içinde keskin bir adım yanıtı (step response) yaratabilir. Ayrıştırıcı, malzemenin Young Modülünü (Elastisite) veya Akma Noktasını (Yield Point) hesaplayacak bir makine öğrenmesi ardışık düzenine (pipeline) veri sağlarken, bu tür sensör anahtarlama anlarını meta verilerden okuyarak temizlemeli veya işaretlemelidir (flagging).Aynı şekilde, MTS TestSuite gibi sistemler, çekme testini "Run", "Hold", "Return" gibi prosedürel aşamalarla (procedure activities) bir iş akışı olarak kaydederler. Dışa aktarılan verinin içinde, rampaların (Ramp(s)) değişimi, yorulma testlerindeki çevrimlerin indeksleri veya zaman kolonundaki ani sıfırlamalar veri setini parçalayabilir. Üretici-bağımsız bir yazılımın en son aşaması, farklı terminolojilerden okunan ve veri tipleri (data types) düzeltilen bu kanalların anlamsal (semantic) olarak yapılandırılmış standart bir JSON şemasına (schema) dökülmesini sağlamaktır.Sonuç itibarıyla, küresel malzeme test cihazlarının birbirinden tamamen bağımsız, dilsel ve yerel idiosenkrazi barındıran CSV ve ASCII dosya yapıları; tasarlanan esnek Strateji sınıfları (Strategy classes), Regex tabanlı Fingerprint algoritmaları ve karakter kodlaması önsezileri (encoding heuristics) sayesinde standart ve yönetilebilir veri havuzlarına dönüştürülebilir. Zaman (Time), Kuvvet (Force), Uzama (Displacement), Gerilme (Stress) ve Birim Şekildeğiştirme (Strain) gibi endüstrinin birleştirici terminolojisinin anahtar rol oynadığı bu haritalama mekanizmaları, laboratuvar otomasyonu ve büyük veri entegrasyonu için yegane sürdürülebilir yoldur.
